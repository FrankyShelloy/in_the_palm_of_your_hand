# Руководство по синхронизации с GitHub

Это руководство поможет вам синхронизировать ваши локальные изменения с репозиторием на GitHub.

## Быстрая синхронизация

### 1. Получение последних изменений с GitHub

```bash
# Получить все изменения с удаленного репозитория
git fetch origin

# Посмотреть, какие изменения есть
git log HEAD..origin/main --oneline

# Влить изменения в вашу текущую ветку
git pull origin main
```

### 2. Отправка ваших изменений на GitHub

```bash
# Добавить все измененные файлы
git add .

# Создать коммит с описанием изменений
git commit -m "Описание ваших изменений"

# Отправить изменения на GitHub
git push origin main
```

## Подробные инструкции

### Проверка статуса

Всегда начинайте с проверки текущего состояния:

```bash
# Проверить состояние локального репозитория
git status

# Посмотреть текущую ветку
git branch

# Посмотреть настроенные удаленные репозитории
git remote -v
```

### Синхронизация с основной веткой (main)

Если вы работаете в своей ветке и хотите синхронизировать её с основной:

```bash
# 1. Сохраните все текущие изменения
git add .
git commit -m "Сохранение текущих изменений"

# 2. Переключитесь на основную ветку
git checkout main

# 3. Получите последние изменения
git pull origin main

# 4. Вернитесь в вашу ветку
git checkout your-branch-name

# 5. Влейте изменения из main в вашу ветку
git merge main
```

### Решение конфликтов

Если при слиянии возникли конфликты:

```bash
# 1. Посмотрите файлы с конфликтами
git status

# 2. Откройте файлы и разрешите конфликты вручную
# Найдите секции с маркерами <<<<<<<, =======, >>>>>>>
# Удалите маркеры и оставьте нужный код

# 3. После разрешения конфликтов добавьте файлы
git add resolved-file.cs

# 4. Завершите слияние
git commit -m "Разрешение конфликтов при слиянии"
```

### Использование скрипта синхронизации

Для упрощения процесса используйте скрипт `sync.sh`:

```bash
# Сделать скрипт исполняемым (один раз)
chmod +x sync.sh

# Запустить синхронизацию
./sync.sh
```

## Рабочий процесс с ветками

### Создание новой ветки для работы

```bash
# Создать и переключиться на новую ветку
git checkout -b feature/new-feature

# Работайте и делайте коммиты
git add .
git commit -m "Добавлена новая функция"

# Отправьте ветку на GitHub
git push -u origin feature/new-feature
```

### Обновление вашей ветки

```bash
# Получить все изменения
git fetch origin

# Влить изменения из main
git merge origin/main

# Или использовать rebase для более чистой истории
git rebase origin/main
```

## Полезные команды

### Просмотр истории

```bash
# Посмотреть последние коммиты
git log --oneline -10

# Посмотреть график веток
git log --graph --oneline --all

# Посмотреть изменения в файле
git log -p filename.cs
```

### Отмена изменений

```bash
# Отменить изменения в файле (до коммита)
git checkout -- filename.cs

# Убрать файл из индекса (после git add)
git reset HEAD filename.cs

# Отменить последний коммит (сохранить изменения)
git reset --soft HEAD~1

# Отменить последний коммит (удалить изменения)
git reset --hard HEAD~1
```

### Временное сохранение изменений

```bash
# Сохранить текущие изменения в stash
git stash

# Посмотреть список сохраненных изменений
git stash list

# Восстановить последние сохраненные изменения
git stash pop

# Применить конкретное сохранение
git stash apply stash@{0}
```

## Настройка репозитория

### Настройка имени и email

```bash
git config user.name "Ваше Имя"
git config user.email "your.email@example.com"
```

### Настройка редактора по умолчанию

```bash
# Для VS Code
git config --global core.editor "code --wait"

# Для nano
git config --global core.editor "nano"
```

## Автоматическая синхронизация

Вы можете настроить автоматическую синхронизацию с помощью:

1. **Git hooks** - автоматические скрипты при определенных событиях
2. **Cron jobs** - периодическая синхронизация по расписанию
3. **GitHub Actions** - автоматизация через CI/CD

Примеры настроек находятся в файле `.github/workflows/sync.yml` (если создан).

## Советы и лучшие практики

1. **Делайте коммиты часто** - маленькие коммиты легче понять и откатить
2. **Пишите понятные сообщения** - объясняйте, что и почему изменили
3. **Синхронизируйтесь регулярно** - получайте изменения от других как можно чаще
4. **Используйте ветки** - не работайте напрямую в main
5. **Проверяйте перед push** - убедитесь, что код работает
6. **Делайте backup** - важные изменения дублируйте

## Troubleshooting

### "Your branch is behind"

```bash
git pull origin main
```

### "Your branch is ahead"

```bash
git push origin main
```

### "Merge conflict"

Следуйте инструкциям в разделе "Решение конфликтов" выше.

### "Permission denied"

Проверьте настройки SSH или используйте HTTPS для клонирования.
